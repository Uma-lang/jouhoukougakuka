JavaScriptのプライベートフィールド（変数）

JavaScript のクラス構文では、変数の前に # を付けることで、その変数を プライベートフィールド（プライベート変数）として定義できます。これにより、クラス外部から直接アクセスできなくなり、データの保護が可能になります。

プライベートフィールドの定義例

class MyClass {
  // プライベートフィールドの宣言
  #data2;

  // コンストラクタで初期化
  constructor(value) {
    this.#data2 = value;
  }

  // プライベートフィールドにアクセスするためのメソッド
  getData() {
    return this.#data2;
  }

  // プライベートフィールドを更新するメソッド
  setData(value) {
    this.#data2 = value;
  }
}

// インスタンスの生成
const myObject = new MyClass(10);

console.log(myObject.getData()); // 10 を表示

myObject.setData(100); // プライベートフィールドの値を更新

console.log(myObject.getData()); // 100 を表示

// クラスの外部から直接アクセスしようとするとエラー
console.log(myObject.#data2); // SyntaxError: Private field '#data2' must be declared in an enclosing class

コードの説明

  • #data2 のように、変数に # を付けることでプライベートフィールドとして宣言します。
  • コンストラクタでプライベートフィールドに値を代入します。
  • getData() というメソッドで、プライベートフィールドの値を外部から取得できます。
  • setData() というメソッドで、プライベートフィールドの値を外部から変更できます。
  • クラス外部から直接プライベートフィールドにアクセスしようとすると、エラー（SyntaxError）が発生します。

プライベートフィールドの利点

  • データの保護: クラス外から直接アクセスできないため、重要なデータを保護できます。
  • カプセル化: クラス内部でデータを管理し、外部には操作用のメソッドを提供することで、データ管理が容易になります。

プライベートフィールドの注意点

プライベートフィールドは、クラス内でしか使用できないため、クラス外からの直接操作が必要な場合には公開メソッドを用意する必要があります。
